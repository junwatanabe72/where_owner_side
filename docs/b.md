# 開発スケジュール詳細（teishaku.md 対応）

## 前提条件
- キックオフ：2025-10-27（月）。
- スプリント：2週間サイクル、フロント1名／バック1名／プロダクト・UX1名体制。
- RESTチャットは既存API改修で構築、評価シミュレーターは新規ドメインモジュールで提供。
- 現行インフラを継続利用し、Node.js／Tailwindスタックを想定。
- UIファースト方針でモックデータを活用した画面先行実装後にAPI・計算ロジックを統合。

## フェーズ別計画

### フェーズ1：UX定義とUIプロトタイピング（2025-10-27 〜 2025-11-07）
- `docs/teishaku.md` の要件粒度を確認し、ユーザーフローと情報設計を更新。
- 評価タブ・チャット・資産台帳の主要画面についてハイレベルモックを作成。
- Tailwindベースのデザインシステム（カラー、タイポ、コンポーネントガイド）を定義。
- 不要機能撤去後のUI空白や遷移影響を整理し、UIファースト遷移案を確定。

### フェーズ2：フロントUI骨格実装（2025-11-10 〜 2025-11-21）
- モックデータを用いてReact UIを構築（評価タブ・3シナリオ比較・チャットスレッド）。
- StorybookやPlayroomでUIコンポーネントを検証し、PM/ステークホルダーがハンズオンで確認。
- Tailwind基調のスタイリングを適用し、MUI依存コンポーネントの置換を開始。
- 登記監視UIとWebSocket UI要素をフラグで無効化し、画面レイアウトを整理。

### フェーズ3：バックエンド契約とAPIスタブ統合（2025-11-24 〜 2025-12-05）
- RESTチャットAPI仕様を確定し、スタブ（モックサーバーまたはSwagger）を提供。
- フロントUIへスタブAPIを接続し、ETagやポーリング制御を擬似実装。
- 評価シミュレーター計算モジュールのインターフェースを定義し、ダミー計算を実装。
- `.env` 供給方式を整理し、秘匿情報の取り扱いに合わせてフロント設定を見直し。

### フェーズ4：機能ロジック実装と不要機能撤去完了（2025-12-08 〜 2025-12-19）
- `/threads` `/messages` APIを本実装し、既存権限ロジックと整合を取る。
- `calc/leaseholdCondo` `calc/landSwap` `calc/masterLease` を本実装し単体テストを整備。
- フロントUIを実データと接続し、未読バッジや評価結果など状態管理を統合。
- 登記監視・WebSocket/SSEコードを削除し、Tailwindへの完全移行とCI更新を行う。

### フェーズ5：総合テストとリリース準備（2025-12-22 〜 2026-01-09）
- 機能テスト、E2Eテスト、チャットポーリングとシミュレーター計算の負荷テストを実施。
- プロダクトデモ、UX微調整、アクセシビリティとレスポンシブ動作を確認。
- README、`docs/teishaku.md` の補足、UIガイドなどドキュメントを更新。
- ステークホルダー検証、リリースノート作成、運用ハンドオフを実施し、年末年始に1週間のバッファを確保。

## リスクと対応
- 税制・金融前提の調整が必要な場合はフェーズ3で税務・法務レビュータスクを追加する。
- `.env` 再編に伴うCI/CD失敗に備え、フェーズ4冒頭でCI更新用のバッファを確保する。
- 年末年始の工数減を見込み、フェーズ5に1週間の調整バッファを確保し機能フリーズ日を厳守する。
- チャットポーリング負荷はフェーズ3で計画を策定し、フェーズ5で検証する。
