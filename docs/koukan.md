承知しました。まずは**「バックエンド開発を行わず、Figma等のデザインツールや、React等のフロントエンドコードのみ」**で動く、**UI/UX検証用のPoC（Proof of Concept）設計**を提案します。

このPoCの目的は、投資家や実際の地主に見せたときに**「これなら自分の資産状況がわかる」「これなら相談に行きやすい」**と実感してもらうことです。

# WHERE PoC設計書（UI検証・静的データ版）

## 1. PoCのコンセプト設定

* **検証シナリオ（ペルソナ）:** 都内に土地を3箇所持つ地主「田中 健一（仮）」
* **ストーリー:** 相続への不安があるが、具体的にどの土地が問題かわからない。WHEREにログインし、状況を把握し、税理士に渡す資料をワンクリックで作るまで。
* **実装範囲:** 主要4画面のみ（遷移可能にする）。

---

## 2. 画面構成（Screen Flow）

1. **【Dashboard】資産一覧・ポートフォリオ**（現状の可視化）
2. **【Asset Detail】個別物件詳細**（課題の特定）
3. **【Consultation Mode】相談パッケージプレビュー**（解決アクション）
4. **【Offer Box】提案一覧**（未来の可能性）

---

## 3. 各画面の詳細設計と静的データ

### ① Dashboard（資産ポートフォリオ）

**目的:** 一目で「資産はあるがお金がない」「対策が必要」な状態を直感させる。

* **UIレイアウト:**
* **上部:** 総資産サマリー（カード表示）
* 推定時価総額：**5.2億円**
* 相続税評価額：**3.8億円**
* 想定相続税額：**1.1億円**（赤字でアラート表示）
* 手元流動性（現預金）：**2,000万円**（「納税資金不足」アイコンを表示）


* **左側:** 地図表示（Google Map風の画像にピンを3つ立てる）
* **右側:** 保有資産リスト（カード型）
1. **自宅（世田谷区）:** 評価額高 / 収益ゼロ
2. **古アパート（練馬区）:** 築45年 / 利回り1.2% / **「修繕リスクあり」**タグ
3. **青空駐車場（埼玉県）:** 稼働率低 / **「利用効率低」**タグ





### ② Asset Detail（課題物件：練馬区 古アパート）

**目的:** なぜその物件が「お荷物」なのか、客観データで突きつける。

* **UIレイアウト:**
* **ヘッダー:** 物件写真、所在地、面積（200㎡）
* **スペック指標（レーダーチャート等で表現）:**
* **収益性:** ★☆☆☆☆ (実質利回り 1.2%)
* **流動性:** ★★☆☆☆ (前面道路狭い)
* **相続評価:** ★★★★★ (減額効果ありだが、収益が見合わない)


* **自動診断コメント（静的テキスト）:**
> 「築45年経過しており、今後5年以内に大規模修繕（約1,500万円）が発生する確率が高いです。現在のキャッシュフローでは賄えないため、売却または等価交換の検討推奨です。」


* **アクションボタン:** 「この物件について専門家に相談する（パッケージ作成）」



### ③ Consultation Mode（相談パッケージ生成）

**目的:** 「これさえあれば手ぶらで相談に行ける」という体験（マジック）を見せる。

* **UIレイアウト:**
* **画面中央:** 生成されたレポートのプレビュー（PDFイメージ）
* **レポート構成要素（モックデータ）:**
* **【A】基礎情報:** 登記簿情報、公図、路線価図（画像配置）
* **【B】課題サマリー:** 「修繕予備費不足」「境界非確定」
* **【C】オーナー意向:** 「先祖代々の土地なので残したいが、手出しはしたくない」
* **【D】不足資料リスト:** 「※ 建築確認済証が見当たりません。探してください」


* **操作UI:**
* 「宛先を選ぶ（税理士・銀行）」→ プルダウン（選択しても動かない）
* 「ダウンロード / 送信」ボタン





### ④ Offer Box（逆指名オファー一覧）

**目的:** 営業電話ではなく「整理された提案」が届く快適さを伝える。

* **UIレイアウト:**
* **ステータス表示:** 「練馬区アパート：組替え検討中」として公開中
* **提案カード（横並び比較）:**
* **提案A（大手不動産）:**
* 種別：**等価交換**
* 提案：マンションへ建替え＋区分所有
* メリット：手出し資金ゼロ、資産性向上
* デメリット：土地所有権の共有化


* **提案B（買取業者）:**
* 種別：**単純売却**
* 金額：**8,500万円**
* メリット：即現金化、納税資金確保
* デメリット：資産総額の減少

## 5. PoC作成ツールのおすすめ

今回は「動く画面」を早く作ることが目的ですので、以下のいずれかをお勧めします。

1. **Figma (デザインプロトタイプ)**
* **メリット:** コードを書かずに「クリックしたら画面遷移」等の動きを作れる。最も速い。
* **適正:** デザインやUXフローを確認したい場合。
* **成果物:** 共有URL（スマホでも操作可能）。


2. **Next.js / React + Tailwind CSS (フロントエンド実装)**
* **メリット:** 実際のWebアプリのような操作感。後でバックエンドを繋げば本番に近づく。
* **適正:** 「ブラウザで実際に動くもの」を見せたい場合。
* **データ:** 上記のJSONをファイルとして読み込ませればOK。
